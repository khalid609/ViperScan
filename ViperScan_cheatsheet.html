ViperScan / Cheat Sheet

v1.0 Python 3.x Threaded Port Scanner Banner Grabber Report Generator
ViperScan is a threaded network recon tool — it discovers live hosts, scans 68 common ports in parallel, grabs service banners to identify running software, detects the OS, then generates a full HTML + TXT report. All three phases run automatically in sequence with a single command.
Execution Flow
Phase 1
🔍
Host Discovery
Builds the target IP list, then scans every IP across 68 common ports using parallel threads.
Resolve domain → IP (if --domain used)
Build IP list based on --mode
Launch one thread per IP
TCP connect_ex() per port (0.5s timeout)
Live progress bar updates in real time
Save open ports → Json/result.json
›
Phase 2
🎯
Banner Scanning
Connects to each open port and tries a series of protocol probes to pull the service banner.
Load hosts from result.json
One thread per host (parallel)
Try 15 protocol probes per port
Decode banner → detect Service + OS
Print result block per host as it finishes
Overwrite result.json with enriched data
›
Phase 3
📊
Report Generation
Reads the enriched JSON and produces two timestamped report files in the Report/ folder.
Load enriched result.json
Calculate risk per port (HIGH/MEDIUM/LOW)
Build stat cards + summary tables
Generate report_YYYYMMDD_HHMMSS.txt
Generate report_YYYYMMDD_HHMMSS.html
Print paths to both files
Scan Modes
--mode single
Scans exactly one IP address. This is the default if no --mode is given. Best for checking a specific machine.
python ViperScan.py --ip 192.168.1.5
python ViperScan.py --domain example.com
Targets: 1 IP, 68 ports = 68 checks
--mode multi
Scans the entire .0–.254 range of the same /24. Use --start and --end to narrow the range. Good for quick LAN sweeps.
python ViperScan.py --ip 192.168.1.1 --mode multi
python ViperScan.py --ip 192.168.1.1 --mode multi --start 10 --end 50
Targets: up to 254 IPs × 68 ports = 17,272 checks
--mode subnet
Uses the subnet calculator to derive the full IP range from a CIDR mask. Requires --subnetmask (8/16/24/32).
python ViperScan.py --ip 192.168.1.1 --mode subnet --subnetmask 24
python ViperScan.py --domain myserver.com --mode subnet --subnetmask 24
Targets: calculated from mask — see subnet table below
Commands & Flags
🖥 Common Commands
Scan a single IP (default mode)
$ python ViperScan.py --ip 192.168.1.1
Scan a domain name
$ python ViperScan.py --domain example.com
Domain is auto-resolved to IP before scanning starts.
Scan full /24 subnet
$ python ViperScan.py --ip 192.168.1.1 --mode subnet --subnetmask 24
Scans 192.168.1.1 → 192.168.1.254 (254 hosts).
Multi mode — full /24 range
$ python ViperScan.py --ip 192.168.1.1 --mode multi
Multi mode — custom last-octet range
$ python ViperScan.py --ip 192.168.1.1 --mode multi --start 10 --end 50
Scans 192.168.1.10 → 192.168.1.50 only.
Faster scan — reduce banner timeout
$ python ViperScan.py --ip 192.168.1.1 --timeout 2
Default is 5s. Use 2s for fast LANs, keep 5s+ for WAN targets.
Rebuild report without rescanning
$ python ViperScan.py --report-only
Reads the existing Json/result.json and regenerates both report files.
Full example — subnet + custom timeout
$ python ViperScan.py --ip 10.0.0.1 --mode subnet --subnetmask 24 --timeout 3
🏷 All Flags Reference
FlagTypeDefaultDescription
--ip str "" Target IPv4 address. Use this OR --domain, not both.
--domain str "" Target domain name. Auto-resolved to IP via DNS before scanning.
--mode str single Scan mode: single multi subnet
--start int 0 Last-octet start index for multi mode. E.g. 10 → starts at x.x.x.10
--end int 255 Last-octet end index for multi mode. E.g. 50 → ends at x.x.x.50
--subnetmask int — CIDR prefix for subnet mode. Must be 8 16 24 or 32
--timeout int 5 Seconds to wait per port when grabbing banners in Phase 2.
--report-only flag off Skip Phases 1 & 2 entirely. Just regenerate reports from existing JSON.

Tips, Tricks & Known Behaviours
⚡
Threading makes it fast
Every IP runs in its own thread during Phase 1 — all ports on all IPs are checked simultaneously. A /24 scan (254 hosts × 68 ports = 17,272 checks) typically completes in under 30 seconds on a local network. Phase 2 also threads one thread per host.
📊
Progress bar during Phase 1
Instead of printing one line per port (which floods the terminal), Phase 1 shows a single live-updating progress bar: [=============> ] 68% | 4,624/6,800 checks | 3 host(s) found. After scanning finishes, a clean summary table is printed.
🕐
Timeout tuning
Phase 1 uses a hardcoded 0.5s socket timeout per port (fast). Phase 2 uses --timeout (default 5s) for banner grabbing. For LAN targets try --timeout 2. For slow or remote WAN hosts keep it at 5 or higher to avoid missed banners.
🌐
Cloudflare & reverse proxies
If banners return Server: cloudflare or Proxy-Status: Cloudflare-Proxy, the real origin server is hidden. OS will show as Unknown — this is expected and correct behaviour. The scan is hitting the CDN edge, not the actual host.
🔁
--report-only workflow
If you already ran a scan, you can edit Report.py (change colours, add fields, tweak layout) and run python ViperScan.py --report-only to regenerate both report files instantly without rescanning. The JSON data is preserved between runs.
🎯
Banner probe order
Phase 2 tries 15 protocol commands per port in order: listen first, then HTTP GET, HTTP HEAD, HTTP/1.1, FTP QUIT, FTP USER, FTP SYST, FTP LIST, SMTP HELP, SMTP AUTH, SMTP NOOP, generic VERSION, SSH handshake, IMAP AUTH, IMAP CAPABILITY. It stops at the first response that returns data.
🔒
Legal reminder
Only scan networks and IP ranges you own or have explicit written permission to test. Unauthorized port scanning is illegal in most countries and may violate ISP terms of service. Never run /8 or /16 scans against public internet IPs.
